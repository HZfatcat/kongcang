### 一、项目概述

**客服运营管理后台**，用于查看客服业务核心 KPI，解决大范围查询性能问题。

**核心业务价值**：
- 从第三方系统同步会话数据到本地数据库，避免直接查询远程系统
- 计算 KPI 指标：用户满意度、需求工单完成率
- 提供可视化 Dashboard 展示

---

### 三、数据模型

```
UdescSession (IM 会话)
├── id, agentId, startedAt, endedAt
├── rating (满意度评分)
├── isConsultToDemand (是否转需求)
└── messages[] → UdescSessionMessage

UdescSessionMessage (会话消息明细)
├── id, sessionId, sentAt
├── senderType, senderId, content
└── rawPayload

ZouwuRequirement (需求工单)
├── id, title, sourceSessionId
├── status (OPEN/IN_PROGRESS/DONE/CLOSED/REJECTED)
├── createdAtSource, completedAtSource
└── rawPayload

SyncCheckpoint (同步断点)
├── source, cursor, lastSyncedAt

SyncRun (同步记录)
├── source, status, recordsSynced

SyncIssue (同步异常)
├── source, category, errorMessage
```

### 四、后端模块

| 模块 | 职责 |
|------|------|
| **HealthModule** | 健康检查接口 |
| **KpiModule** | KPI 计算：满意度率、需求完成率 |
| **SyncModule** | 数据同步调度与执行 |
| **UdescModule** | Udesk 数据查询 API |

**数据同步流程**：

```
┌──────────────────────────────────────────────────────────────┐
│                       syncUdesc()                            │
├──────────────────────────────────────────────────────────────┤
│  时间窗口循环 ──────────────────────────────────────────────► │
│   │                                                          │
│   ├─► fetchSessions() ─► upsert UdescSession                 │
│   │        │                                                 │
│   │        └─► fetchSessionLogs() ─► upsert UdescSessionMessage │
│   │                                                          │
│   └─► 更新 SyncCheckpoint                                    │
│                                                              │
│  失败记录 ─► SyncIssue                                       │
└──────────────────────────────────────────────────────────────┘
```

**关键特性**：
- 时间窗口分片同步（避免一次性拉取大量数据）
- 重试机制 (`withRetry`)
- 幂等 upsert（避免重复数据）
- 断点续传
- 失败问题记录

### 七、第三方集成

#### Udesk（客服系统）
- **认证**：OpenAPI v2 签名 (email + token + timestamp + nonce + sha256)
- **同步入口**：
  - `im/sessions/search` → 会话列表
  - `im/sessions/vote` → 满意度评分
  - `im/sessions/log` → 消息明细

#### 驺吾（需求管理系统）
- **状态**：可选启用 (`SYNC_ENABLE_ZOUWU`)
- **同步**：需求工单及其状态流转

### 十、架构亮点

1. **数据本地化**：定期同步第三方数据到本地，解决大范围查询性能瓶颈
2. **幂等设计**：所有写入使用 upsert，支持重复同步
3. **断点续传**：记录同步游标，支持中断后继续
4. **容错机制**：失败不影响整体，问题记录到 SyncIssue 表
5. **签名鉴权**：完整实现 Udesk OpenAPI v2 签名机制
6. **字段兼容**：多候选字段映射，适配不同数据源结构
